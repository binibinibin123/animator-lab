# AutoVideo UI/UX 전면 개편 명세서

버전: 1.0  
작성일: 2026-02-24  
대상: 디자인/프론트엔드 구현 공통 기준 문서

---

## 1) 서비스 개요 (이 앱이 무엇인지)

AutoVideo는 주제 입력만으로 숏폼 영상을 제작하는 AI 기반 제작 앱이다.  
현재 핵심 흐름은 다음과 같다.

1. Script 생성
2. TTS 생성
3. Image 생성
4. Video 생성
5. 최종 Render/Download

앱의 차별점은 "생성 파이프라인을 한 화면 흐름에서 제어"하는 점이며, SaaS 전환 시점에서는 여기에 "크레딧 기반 사용량 UX"를 명확히 결합해야 한다.

---

## 2) 개편 목표

### 제품 목표
- 기존 기능은 유지하면서 정보구조(IA)와 화면 역할을 SaaS 운영형으로 재설계한다.
- 사용자에게 비용(크레딧), 상태(진행률/실패), 다음 행동(CTA)을 즉시 이해시키는 UX를 만든다.

### UX 목표
- 로그인 진입은 페이지 이동이 아닌 모달 기반으로 처리한다.
- 페이지 역할을 명확히 분리한다.
  - 랜딩: 획득/전환
  - 대시보드: 운영/관리
  - 편집 플로우: 생산/실행
- 모든 생성 액션에 예상 크레딧/실사용량 문맥을 붙인다.

---

## 3) 설계 원칙

1. 기능 우선 + 비용 가시성 우선
2. 페이지별 단일 목적 유지
3. 상태 표현 일관성 유지 (queued/running/completed/failed/cancelled)
4. 실패 메시지는 "문제 + 다음 행동" 구조
5. 모바일/데스크톱 모두 동일한 정보 우선순위 제공

---

## 4) 크레딧 표기 정책 (UI 필수 반영)

### 기준값
- 30초 영상 1편: 730 credits
- 60초 영상 1편: 1,325 credits

### 단계 단가
- Script: 50 / 회
- TTS: 15 / 세그먼트
- Image: 25 / 세그먼트
- Video: 45 / 세그먼트

### UI 노출 규칙
- 실행 버튼 근처: "예상 소모" 필수
- 실행 완료 후: "실제 차감" 표기
- 상단 영역: "잔여 크레딧" 고정 노출

---

## 5) 정보 구조 (IA)

### A. 랜딩 `/`
역할: 유입/설득/로그인 전환

필수 섹션:
- Hero (가치 제안)
- 파이프라인 요약 (Script→TTS→Image→Video→Render)
- 크레딧 가이드 (30초/60초)
- 로그인 CTA (모달 오픈)

금지:
- 대시보드 기능을 랜딩에 섞지 않음

### B. 프로젝트 대시보드 `/projects`
역할: 제작 운영 허브

필수 섹션:
- 사용자/플랜/잔여 크레딧
- 빠른 시작(온보딩)
- 프로젝트 목록(검색/정렬/필터)
- 새 프로젝트/오토파일럿 CTA

필수 카드 정보:
- 제목, 상태, 썸네일, 생성일, 비율
- 다음 단계 이동 CTA
- 예상 추가 소모(가능하면)

### C. 편집 플로우 `/create/*`, `/project/[id]/*`
역할: 실제 제작 실행

필수 구성:
- 상단: 현재 단계 + 예상 소모 + 잔여 크레딧 컨텍스트
- 스테퍼: 현재/완료/다음 단계
- 본문: 단계별 생성 액션
- 로그/진행률: 실시간 작업 피드백

## 6) 인증 UX 명세

### 로그인 방식
- 로그인 전용 페이지가 아니라 "전역 모달"을 기본 진입으로 사용
- 지원 provider: Google, Naver, Kakao

### 동작 규칙
- 랜딩 CTA 클릭 -> 로그인 모달 오픈
- 보호 라우트 접근 미인증 시 -> 랜딩으로 이동 + 모달 자동 오픈
- 로그인 성공 시 -> 원래 의도 경로(없으면 `/projects`) 복귀

### 오류 처리
- provider 취소/거절: 모달 내 안내 메시지
- 콜백 실패: 재시도 버튼 + fallback provider 제시

---

## 7) 페이지별 상세 UI 요구사항

### 7.1 랜딩
- 첫 화면에서 5초 내 아래 3개를 이해 가능해야 함:
  1) 어떤 제품인지
  2) 어떤 흐름으로 작동하는지
  3) 비용이 어떻게 계산되는지

### 7.2 대시보드
- 상단 KPI 최소 3개:
  - 잔여 크레딧
  - 월 사용량
  - 완료율
- 프로젝트가 없는 경우도 "다음 행동"이 명확해야 함

### 7.3 Script 단계
- 생성 버튼에 Script 비용 노출
- 생성 전/후 상태 구분 시각화

### 7.4 Voice/Image/Video 단계
- 세그먼트 수 기반 예상 소모 자동 표기
- 실패 세그먼트만 재시도 가능한 UX 우선

### 7.5 Preview/Render 단계
- 렌더 상태, 예상 시간, 완료 후 다운로드 경로를 명확히 표시
- 실패 시 단순 실패가 아니라 재시도 액션 즉시 제공

---

## 8) 비주얼 방향 (재브랜딩 가이드)

### 방향성
- 기존 브랜드 문맥(보라 계열, 라운드, 생산도구 느낌)을 유지하면서 완성도만 상향
- "완전 다른 제품처럼" 보이는 파격 리브랜딩은 지양

### 타이포/컬러
- 헤드라인은 강한 위계, 본문은 가독성 우선
- 과도한 대비/색 분산 대신 핵심 강조색을 집중 사용

### 모션
- 페이지 진입 시 과한 애니메이션 금지
- 상태 변화(로딩/완료/실패) 중심의 목적형 모션만 허용

---

## 9) 구현 범위 / 비범위

### 범위 포함
- 랜딩/대시보드/편집 플로우 UI 개편
- 로그인 모달 UX
- 크레딧 노출 체계

### 범위 제외
- 결제 시스템 구현
- 생성 엔진 로직 변경
- 대규모 백엔드 아키텍처 변경

---

## 10) 완료 기준 (Definition of Done)

- [ ] 로그인 모달 흐름이 랜딩/보호 라우트 모두에서 일관 동작
- [ ] 3개 페이지군(랜딩/대시보드/편집)의 역할이 명확히 분리
- [ ] 모든 핵심 생성 버튼에서 예상 크레딧 확인 가능
- [ ] 상태/오류 메시지에 다음 행동이 명시됨
- [ ] 모바일/데스크톱에서 핵심 정보가 우선순위대로 보임

---

## 11) 레퍼런스 제작 지시 템플릿 (HTML/이미지 제작용)

각 화면마다 아래 항목을 반드시 포함한다.

1. 화면 목적 (획득/운영/생산 중 무엇인지)
2. 주요 사용자 행동 1~2개
3. 필수 데이터 블록 (크레딧/상태/CTA)
4. 상태 버전 (기본/로딩/오류/완료)
5. 모바일 버전

이 템플릿을 충족하지 않으면 레퍼런스 산출물로 승인하지 않는다.
